# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Kiki OS is an open research project asking how far a fully agentic OS can change human-computer interaction. The deliverable is two things: an OS people can actually run, and observations about agentic HCI that the community can scrutinize. Correctness of the documentation corpus is load-bearing — it is the spec for everything that will be built.

## Repo layout

```
docs/         — all documentation
  00-foundations … 14-rfcs/  — chapter dirs: DESIGNs, GUIDEs, ADRs, RFCs, INDEX files
  specs/        — all SPEC documents (131 files, flat), every file has type: SPEC
  meta/         — auto-generated: DEPENDENCY-GRAPH.md, COVERAGE-MATRIX.md (do not edit by hand)
  CONVENTIONS.md, CONTRIBUTING.md, README.md
.claude/      — agent team config: settings.json, hooks/, agents/
CLAUDE.md     — this file
```

Chapter directories hold DESIGNs, GUIDEs, ADRs, RFCs, and INDEX files only. No SPECs live there.

## Document types and required sections

| Type | Location | Required sections |
|---|---|---|
| **SPEC** | `specs/` | Purpose · Inputs · Outputs · Behavior · Interfaces · State · Failure modes · Performance contracts · Acceptance criteria · Open questions · References |
| **DESIGN** | `docs/NN-chapter/` | Problem · Constraints · Decision · Rationale · Consequences · References |
| **ADR** | `docs/14-rfcs/` | Status · Context · Decision · Consequences · Alternatives considered · References |
| **RFC** | `docs/14-rfcs/` | Summary · Motivation · Detailed design · Drawbacks · Alternatives · Unresolved questions · Adoption strategy |
| **GUIDE** | `docs/NN-chapter/` | Free-form, starts with Purpose |

## Frontmatter (required on every file)

```yaml
---
id: kebab-case-stable-id      # never change after creation
title: Human Title
type: DESIGN | SPEC | GUIDE | ADR | RFC
status: draft                  # stays draft until the subsystem is implemented
version: 0.0.0
implements: []
depends_on: []                 # list of doc IDs (kebab-case), not file paths
depended_on_by: []             # auto-generated by linter — do not author manually
last_updated: YYYY-MM-DD
---
```

`id` is stable forever. Changing it breaks every document that references it via `depends_on`.  
`depended_on_by` is regenerated by the linter on every CI run — do not write it manually.

## Cross-references

- `depends_on` / `depended_on_by` use **kebab-case IDs**, never file paths.
- Prose links in INDEX files use **relative paths**: `../specs/FILENAME.md` for SPECs (when inside a chapter dir), or `specs/FILENAME.md` from the repo root.
- Filenames are always `UPPERCASE-WITH-DASHES.md`.

## When to open an RFC vs. direct PR

Open an RFC first if: the change alters the contract other documents depend on, adds a new chapter, or introduces a cross-cutting capability or constraint. Everything else (new SPEC in an existing chapter, typos, clarifications) goes as a direct PR.

## Authoring rules

- Every SPEC must have a testable `## Acceptance criteria` checklist. A SPEC without it is not done.
- Every DESIGN must state `## Problem` first. A design without a problem is decoration.
- `## Constraints` in DESIGNs rule out alternatives; weak constraints make weak decisions.
- Language: English, American spelling. "the agent" or "Kiki" — not "AI". "user" for the human, never "operator".
- No marketing language (smart, seamless, magical). State what the system does.
- Pseudocode is Rust-flavored. Diagrams: ASCII first, Mermaid only when ASCII does not scale.

## Agent team norms

This repo is configured for Claude Code agent teams (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`).

- No two teammates edit the same file concurrently.
- Mark a task `in_progress` before starting; mark `completed` only after reading and verifying the output.
- Commit after each unit of completed work. Do not batch unrelated changes.
- Do not expand scope beyond what the task states. When in doubt, ask the lead.
- Do not move files without updating all cross-references.
- Do not create files outside `docs/` without explicit instruction.

## Hooks (enforced automatically)

- `PreToolUse(Bash)` — blocks destructive git operations (`--force`, `--hard`, `rm -rf`).
- `TaskCompleted` — blocks task completion if SPEC files exist outside `specs/`.

## Available agent roles (`.claude/agents/`)

- `spec-writer` — drafts and edits SPECs in `specs/`, runs in an isolated worktree.
- `doc-reviewer` — audits docs for convention compliance, read-only.
- `adr-writer` — writes ADRs in `docs/14-rfcs/`, runs in an isolated worktree.
