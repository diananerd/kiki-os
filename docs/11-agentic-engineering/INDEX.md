# Agentic Engineering

Research-validated patterns for agentic systems. Distinct from runtime SPECs because these are cross-cutting heuristics distilled from production agentic systems (Claude Code, Cursor, Devin, Aider, Cline, NemoClaw, OpenAI Codex, etc.) and academic research.

## Patterns

- `HARNESS-PATTERNS.md` — seven-component decomposition and variants.
- `PROMPT-INJECTION-DEFENSE.md` — lethal trifecta, CaMeL, AgentDojo numbers.
- `CONTEXT-ENGINEERING.md` — cache discipline, sticky latches, prefix preservation.
- `MULTI-AGENT-POLICY.md` — when to spawn (research findings on overhead).
- `COST-CONTROL.md` — budgets, circuit breakers, diminishing returns.
- `EVALUATION.md` — benchmarks, regression gates.
- `CURATED-PROMPTS.md` — versioned-as-code prompts: arbiter, compaction, search.
- `MODEL-LIFECYCLE.md` — how models reach the device, integrity verification.
