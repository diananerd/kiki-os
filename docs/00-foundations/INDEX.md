# Foundations

The foundational chapter. Every other chapter depends on these documents. Read in this order:

1. **`VISION.md`** — what Kiki OS is and is not.
2. **`PARADIGM.md`** — the irreducible shape of Kiki OS as an appliance OS for agentic computing. The cornerstone.
3. **`PRINCIPLES.md`** — ordered design principles. Higher-priority principles win conflicts.
4. **`DETERMINISTIC-VS-AGENTIC.md`** — where LLMs live, where deterministic code does. The rule that prevents hallucination from being load-bearing.
5. **`GLOSSARY.md`** — canonical vocabulary. If a term means something specific in Kiki, it is defined here.
6. **`DESIGN-PHILOSOPHY.md`** — anti-patterns, dos and don'ts learned from research and experience.
7. **`ROADMAP.md`** — v0 / v0.5 / v1 phasing.

Documents in this chapter have no `depends_on` — they are the root of the dependency graph.
